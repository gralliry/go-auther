设计方案：项目结构 + 性能优化

高频读取优先 · 子包功能拆分 · 零循环依赖

新项目结构

文件布局（↓ 依赖方向，上层依赖下层）
auther/
├── model/                    ← 数据模型（零依赖）
│   └── model.go              RoleNode, UserNode, RoleGrant, Snapshot, Info
│
├── match/                    ← 匹配引擎（零依赖）
│   ├── match.go              Segment(), Parse(), DPMatch()
│   └── segment.go            segs struct + 方法
│
├── adapters/
│   ├── memory/              内存适配器
│   └── file/                 JSON 文件适配器
│
├── examples/
│   └── basic/
│       └── main.go           基本使用示例
│
├── auther.go                 Authorizer + NewAuthorizer + 树内部方法
├── enforce.go                Enforce / GetUserPermissions
├── grant.go                  Grant / Revoke / 授权查询
├── role.go                   角色 CRUD + 查询
├── user.go                   用户 CRUD + 查询
├── resource.go               normalizeResource（调用 match 包）
├── errors.go                 哨兵错误定义
├── adapter.go                Adapter 接口
├── go.mod
└── README.md

优点

  • match 和 model 零依赖，可独立测试和复用
  • 无循环依赖：model ← match ← root
  • 外部只需 import "auther"，子包按需导入
  • examples/ 提供标准参考实现

代价

  • model.RoleGrant 等类型引用路径变长
  • 需要在 root 包中用 type alias 简化常用类型

性能优化方案（按收益排序）

#优化项技术手段预期收益
1 match 零分配 预分配 segs 缓存 + 迭代匹配替代 DP 表分配 Match 从 300ns → 50ns
2 Grants 索引 GrantsIn 从切片改为 map[resource]bool，O(1) 查找 HitGrant 从 719ns → 400ns
3 资源缓存 Enforce 对同一角色+资源组合缓存匹配结果 重复检查 → 50ns
4 sync.Map 替代 RWMutex 对 roles/users map 使用分段锁或 sync.Map 读延迟降低 30%
5 write-through → write-back 写操作异步批量刷盘，不阻塞读 写操作延迟降低 90%
